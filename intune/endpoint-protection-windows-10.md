# <a name="windows-10-and-later-settings-to-protect-devices-using-intune"></a>Windows 10（及更高版本）设置，用于保护使用 Intune 的设备

[!INCLUDE [azure_portal](./includes/azure_portal.md)]

Microsoft Intune 包括许多设置，可帮助保护设备。 本文介绍可以在 Windows 10 和更高版本的设备中启用和配置的所有设置。 这些设置是在 Intune 中的 Endpoint Protection 配置文件中创建的，用于控制安全性，包括 BitLocker 和 Windows Defender。

若要配置 Windows Defender 防病毒软件，请参阅 [Windows 10 设备限制](device-restrictions-windows-10.md#windows-defender-antivirus)。

## <a name="before-you-begin"></a>在开始之前

[创建 Endpoint Protection 设备配置配置文件](endpoint-protection-configure.md)。

## <a name="windows-defender-application-guard"></a>Windows Defender 应用程序防护

在以下 Windows 10 版本中受支持：

- 企业
- 专业

使用 Microsoft Edge 时，Windows Defender 应用程序防护可保护环境免受组织不信任的站点的影响。 用户访问独立网络边界中未列出的站点时，这些站点将在 Hyper-V 虚拟浏览会话中打开。 受信任的站点由在设备配置中配置的网络边界定义。

应用程序防护仅适用于 Windows 10（64 位）设备。 使用此配置文件将安装用于激活应用程序防护的 Win32 组件。

- **应用程序防护**：“启用”可打开此功能，在 Hyper-V 虚拟化浏览容器中打开未经批准的站点。 “未配置”（默认设置）意味着在该设备上打开所有站点（已批准和未批准的）。
- **剪贴板行为**：选择允许在本地电脑和应用程序防护虚拟浏览器之间执行的复制/粘贴操作。
- **企业网站上的外部内容**：“阻止”可阻止加载未经批准的网站中的内容。 “未配置”（默认设置）意味着可以在该设备上打开非企业站点。
- **从虚拟浏览器打印**：选择“允许”，使 PDF、XPS、本地和网络打印机能够打印虚拟浏览器中的内容。 “未配置”（默认设置）会禁用所有打印功能。
- **收集日志**：设置为“允许”可收集应用程序防护浏览会话内发生的事件的日志。 “未配置”（默认设置）不会收集浏览会话内的任何日志。
- **保留用户生成的浏览器数据**：设置为“允许”可保存应用程序防护虚拟浏览会话期间创建的用户数据（如密码、收藏和 cookie）。 “未配置”（默认设置）会在设备重启或用户注销时放弃用户下载的文件和数据。
- **图形加速**：设置为“启用”可通过获取对虚拟图形处理单元的访问权限来提升加载图形密集型网站和视频的速度。 “未配置”（默认设置）在处理图形时使用设备 CPU；不使用虚拟图形处理单元。
- **将文件下载到主机文件系统**：设置为“启用”可让用户从虚拟化浏览器将文件下载到主机操作系统。 “未配置”（默认设置）将文件本地保存在设备上，而不会下载到主机文件系统。

## <a name="windows-defender-firewall"></a>Windows Defender 防火墙

在以下 Windows 10 版本中受支持：
- 住宅
- 专业
- 商用版
- 企业
- 教育
- “移动”
- 移动企业版

### <a name="global-settings"></a>全局设置

以下设置适用于所有网络类型。

- **文件传输协议**：设置为“阻止”，可禁用有状态 FTP。 设置为“未配置”（默认设置）时，防火墙会筛选监控状态的 FTP 以允许辅助连接。
- **删除前的安全关联空闲时间**：如果在 n 秒内未检测到任何网络流量，则删除安全关联。 输入以秒为单位的空闲时间。
- **预共享密钥编码**：设置为“启用”，使用采用 UTF-8 的预共享密钥编码。 “未配置”（默认设置）使用本地存储值。
- **IPsec 免除**：将特定流量配置为不受 IPsec 保护，包括：
  - **邻居发现 IPv6 ICMP 类型代码**
  - **ICMP**
  - **路由器发现 IPv6 ICMP 类型代码**
  - **IPv4 和 IPv6 DHCP 流量**
- **证书吊销列表验证**：选择设备验证证书吊销列表的方式。 选项包括“禁用 CRL 验证”、“仅在遇到已吊销的证书时，CRL 验证才失败”和“如果遇到任何错误，则 CRL 验证失败”。
- **适时地按每个密钥模块匹配身份验证集**：如果设置为“启用”，密钥模块必须仅忽略它们不支持的身份验证套件。 设置为“未配置”时，如果密钥模块不支持此集中指定的所有身份验证套件，则必须忽略整个身份验证集。
- **数据包排队**：指定如何针对 IPsec 隧道网关方案为加密接收和明文转发启用接收端上的软件缩放。 此设置可确保数据包顺序得到保留。

### <a name="network-settings"></a>网络设置

以下设置适用于特定网络类型，包括“域(工作区)网络”、“专用(可检测)网络”和“公用(不可检测)网络”。

#### <a name="general-settings"></a>常规设置

- **Windows Defender 防火墙**：设置为“启用”可打开此防火墙并提升安全性。 “未配置”（默认设置）允许所有流量而不考虑任何其他策略设置。
- **隐藏模式**：设置为“阻止”可阻止防火墙在隐藏模式下运行。 阻止隐藏模式还可以阻止“IPsec 安全数据包免除”。 “未配置”（默认设置）可让防火墙在隐藏模式下运行，有助于阻止对探测请求的响应。
- **受防护**：设置为“阻止”可以关闭此功能。 “未配置”（默认设置）会启用此设置。 如果开启此设置和 Windows Defender 防火墙，则会阻止所有传入流量，而不考虑任何其他策略设置。
- **针对多播广播的单播响应**：如果设置为“阻止”，会禁用对多播广播的单播响应。 通常，你不希望接收针对多播或广播消息的单播响应。 这些响应可能表示拒绝服务 (DOS) 攻击，或攻击者尝试探测已知的实时计算机。 “未配置”（默认设置）会启用此设置。
- **入站通知**：设置为“阻止”时，会在阻止应用侦听某个端口时隐藏对用户的通知。 “未配置”（默认设置）可能会在阻止应用侦听某个端口时向用户显示一条通知。
- **入站连接默认操作**：设置为“阻止”时，不会在入站连接上运行默认的防火墙操作。 设置为“未配置”（默认设置）时，会在入站连接上运行默认的防火墙操作。

#### <a name="rule-merging"></a>规则合并

- **来自本地存储的已授权应用程序 Windows Defender 防火墙规则**：设置为“启用”可应用本地存储中的防火墙规则，以便识别和强制执行它们。 设置为“未配置”（默认设置）时，会忽略且不会强制执行本地存储中已授权的应用程序防火墙规则。
- **来自本地存储的全局端口 Windows Defender 防火墙规则**：设置为“启用”可应用本地存储中要识别和强制执行的全局端口防火墙规则。 设置为“未配置”（默认设置）时，会忽略且不会强制执行本地存储中的全局端口防火墙规则。
- **来自本地存储的 Windows Defender 防火墙规则**：设置为“启用”可应用本地存储中要识别和强制执行的防火墙规则。 设置为“未配置”（默认设置）时，会忽略且不会强制执行本地存储中的防火墙规则。
- **来自本地存储的 IPsec 规则**：设置为“启用”可应用本地存储中的连接安全规则，而不考虑架构或连接安全规则版本。 设置为“未配置”（默认设置）时，会忽略且不会强制执行本地存储中的连接安全规则，而不考虑架构版本和连接安全规则版本。

## <a name="windows-defender-smartscreen-settings"></a>Windows Defender SmartScreen 设置

在已安装 Microsoft Edge 的以下 Windows 10 版本中受支持：
- 住宅
- 专业
- 商用版
- 企业
- 教育
- “移动”
- 移动企业版

**设置**：

- **用于应用和文件的 SmartScreen**：设置为“启用”可将 Windows SmartScreen 用于执行文件和运行应用。 SmartScreen 是一种基于云的防网络钓鱼和反恶意软件组件。 “未配置”（默认设置）会禁用 SmartScreen。
- **未经验证的文件执行**：设置为“阻止”可阻止最终用户运行未经 Windows SmartScreen 验证的文件。 “未配置”（默认设置）会禁用此功能，并允许最终用户运行未经验证的文件。

## <a name="windows-encryption"></a>Windows 加密

### <a name="windows-settings"></a>Windows 设置

在以下 Windows 10 版本中受支持：

- 专业
- 商用版
- 企业
- 教育
- “移动”
- 移动企业版

**设置**：

- **加密设备**：选择“需要”以提示用户启用设备加密。 根据不同的 Windows 版本和系统配置，用户可能需要执行不同的操作：  
  - 确认未启用来自其他提供程序的加密功能
  - 需要关闭 Bitlocker 驱动器加密，然后重新开启 Bitlocker
    
    如果 Windows 加密开启时另一种加密方法处于活动状态，设备可能会变得不稳定。 
- **加密存储卡（仅限移动设备）**：选择“需要”可对设备使用的任何可移动存储卡进行加密。 “未配置”（默认设置）不需要存储卡加密，也不会提示用户开启它。 此设置仅适用于 Windows 10 移动设备。

### <a name="bitlocker-base-settings"></a>BitLocker 基本设置

在以下 Windows 10 版本中受支持：

- 企业
- 教育
- “移动”
- 移动企业版
- 专业

基本设置是适合所有数据驱动器类型的通用 BitLocker 设置。 这些设置用于管理最终用户可以在所有类型的数据驱动器中修改哪些驱动器加密任务或配置选项。

- **其他磁盘加密的警告**：选择“阻止”可禁用关于设备上存在其他磁盘加密服务的警告。 “未配置”（默认设置）允许显示此类警告。
    - **允许标准用户在 Azure AD 加入期间启用加密**：如果你选择“允许”，标准用户/非管理员可以在用户登录时启用 BitLocker 加密。 此设置仅适用于 Azure Active Directory 加入 (Azure ADJ) 设备。 “未配置”仅允许管理员在设备上启用 BitLocker 加密。
      
      此设置仅适用于 Azure Active Directory 加入 (Azure ADJ) 设备。 它还要求必须将“针对其他磁盘加密的警告”设置为“阻止”。
- **配置加密方法**：设置为“启用”启用此设置，可配置操作系统、数据和可移动驱动器的加密算法。 设置为“未配置”（默认设置）时，BitLocker 会将 XTS-AES 128 位用作默认加密方法，或使用任何安装脚本指定的加密方法。
  - **操作系统驱动器加密**：选择操作系统驱动器的加密方法。 建议使用 XTS-AES 算法。
  - **固定数据驱动器的加密**：选择固定（内置）数据驱动器的加密方法。 建议使用 XTS-AES 算法。
  - **可移动数据驱动器的加密**：选择可移动数据驱动器的加密方法。 如果在不运行 Windows 10 的设备上使用可移动驱动器，建议使用 AES-CBC 算法。

### <a name="bitlocker-os-drive-settings"></a>BitLocker OS 驱动器设置
在以下 Windows 10 版本中受支持：

- 企业
- 教育
- “移动”
- 移动企业版
- 专业

以下设置仅适用于操作系统数据驱动器。

- **启动时的其他身份验证**：选择“需要”配置计算机启动时的身份验证要求，包括使用受信任的平台模块 (TPM)。 选择“未配置”（默认设置）时，只会使用 TPM 在设备上配置基本选项。
  - **包含非兼容 TPM 芯片的 BitLocker**：在设备不具备兼容 TPM 芯片时“阻止”使用（禁用）BitLocker。 设置为“未配置”时，用户能在不具备兼容 TPM 芯片的情况下使用 BitLocker。 BitLocker 可能需要密码或启动密钥。
  - **兼容的 TPM 启动**：选择是允许、不允许还是要求使用 TPM 芯片。
  - **兼容的 TPM 启动 PIN**：选择是允许、不允许还是要求使用带有 TPM 芯片的启动 PIN。 启用启动 PIN 需要最终用户进行交互。 
  - **兼容的 TPM 启动密钥**：选择是允许、不允许还是要求使用带有 TPM 芯片的启动密钥。 启用启动密钥需要最终用户进行交互。 
  - **兼容的 TPM 启动密钥和 PIN**：选择是允许、不允许还是要求使用带有 TPM 芯片的启动密钥和 PIN。 启用启动密钥和 PIN 需要最终用户进行交互。
- **最小 PIN 长度**：“启用”可启用此设置，以配置 TPM 启动 PIN 的最小长度。 设置为“未配置”（默认设置）时，用户可以配置 6 到 20 位之间任何长度的启动 PIN。
  - **最少字符数**：输入启动 PIN 所需的字符数（介于 4-20）。
- **OS 驱动器恢复**：“启用”此设置可控制在所需启动信息不可用时如何恢复受 BitLocker 保护的操作系统驱动器。 设置为“未配置”（默认设置）时，BitLocker 恢复支持默认的恢复选项。 默认情况下允许使用 DRA，由用户选择恢复选项（包括恢复密码和恢复密钥），并且不会将恢复信息备份至 AD DS。
  - **基于证书的数据恢复代理**：设置为“阻止”时，不能将数据恢复代理用于受 BitLocker 保护的操作系统驱动器。 设置为“未配置”（默认设置）可以启用此设置，让数据恢复代理可用于受 BitLocker 保护的操作系统驱动器。
  - **用户创建恢复密码**：选择是允许、要求还是不允许用户生成 48 位数的恢复密码。
  - **用户创建恢复密钥**：选择是允许、要求还是不允许用户生成 256 位数的恢复密钥。
  - **BitLocker 安装向导中的恢复选项**：设置为“阻止”使用户无法查看和更改恢复选项。 设置为“未配置”（默认设置）时，用户可以在开启 BitLocker 时查看并更改恢复选项。
  - **将 BitLocker 恢复信息保存到 AD DS**：选择“启用”，将 BitLocker 恢复信息存储到 Azure Active Directory (AAD)。 设置为“未配置”（默认设置）时，不会将恢复信息存储在 AAD 中。
  - **存储到 AD DS 的 BitLocker 恢复信息**：配置将 BitLocker 恢复信息的哪些部分存储到 Azure AD 中。 选择：
    - “备份恢复密码和密钥包”
    - “仅备份恢复密码”
  - **在启用 BitLocker 前将恢复信息存储在 AD DS 中**：需要此设置来阻止用户启用 BitLocker，除非 BitLocker 恢复信息已成功存储在 Azure Active Directory (AD) 中。 “未配置”（默认设置）允许用户开启 BitLocker，即使未能成功将恢复信息存储在 Azure AD 中。
- **预启动恢复消息和 URL**：“启用”此设置以配置预启动密钥恢复屏幕上显示的消息和 URL。 “未配置”（默认设置）会禁用此功能。
  - **预启动恢复消息**：配置如何向用户显示预启动恢复消息。 选择：
    - “使用默认恢复消息和 URL”
    - “使用空的恢复消息和 URL”
    - “使用自定义恢复消息”
    - “使用自定义恢复 URL”

### <a name="bitlocker-fixed-data-drive-settings"></a>BitLocker 固定数据驱动器设置

在以下 Windows 10 版本中受支持：

- 企业
- 教育
- “移动”
- 移动企业版
- 专业

**设置**：

- **对不受 BitLocker 保护的固定驱动器的写访问**：设置为“阻止”，对不受 BitLocker 保护的数据驱动器授予只读访问权限。 设置为“未配置”（默认设置）时，则具备对不受 BitLocker 保护的数据驱动器的读取和写入访问权限。
- **固定驱动器恢复**：“启用”此设置可控制在所需启动信息不可用时如何恢复受 BitLocker 保护的固定驱动器。 “未配置”（默认设置）会禁用此功能。
  - **数据恢复代理**：阻止受 BitLocker 保护的固定驱动器策略编辑器使用数据恢复代理。 “未配置”（默认设置）允许将数据恢复代理用于受 BitLocker 保护的固定驱动器。
  - **用户创建恢复密码**：配置是允许、要求还是不允许用户生成 48 位数的恢复密码。  
  - **用户创建恢复密钥**：配置是允许、要求还是不允许用户生成 256 位数的恢复密钥。
  - **BitLocker 安装向导中的恢复选项**：设置为“阻止”使用户无法查看和更改恢复选项。 设置为“未配置”（默认设置）时，用户可以在开启 BitLocker 时查看并更改恢复选项。
  - **将 BitLocker 恢复信息保存到 AD DS**：选择“启用”，将 BitLocker 恢复信息存储在 Azure Active Directory (Azure AD) 中。 设置为“未配置”（默认设置）时，不会将恢复信息存储在 Azure AD 中。
  - **存储到 AD DS 的 BitLocker 恢复信息**：配置将 BitLocker 恢复信息的哪些部分存储到 Azure AD 中。 选项包括：
    - “备份恢复密码和密钥包”
    - “仅备份恢复密码”
  - **在启用 BitLocker 前将恢复信息存储在 AD DS 中**：需要此设置来阻止用户启用 BitLocker，除非 BitLocker 恢复信息已成功存储在 Azure AD 中。 “未配置”（默认设置）允许用户开启 BitLocker，即使未能成功将恢复信息存储在 Azure AD 中。

### <a name="user-account-control"></a>用户帐户控制

- **无安全位置的 UIA 完整性**：设置为“启用”时，位于文件系统中安全位置的应用只按 UIAccess 完整性运行。 “未配置”（默认设置）可让应用按 UIAccess 完整性运行，即使应用不在文件系统中的安全位置。
- **将文件和注册表写入错误虚拟化到每用户位置**：设置为“阻止”时，在运行时将应用程序写入错误重定向到文件系统和注册表的已定义用户位置。 设置为“未配置”（默认设置）时，将数据写入受保护位置的应用程序将会无法执行。
- **只提升已签名并验证的可执行文件**：设置为“启用”可在运行可执行文件之前，强制对其执行 PKI 证书路径验证。 设置为“未配置”（默认设置）时，在某个可执行文件可以运行之前不会强制对其执行 PKI 证书路径验证。

- **管理员的提升提示**：在管理员批准模式下定义管理员的提升提示行为：

  - 在安全桌面上同意提示
  - 提示凭据
  - 同意提示
  - **未配置**：非 Windows 二进制文件的同意提示
- **标准用户的提升提示**：定义标准用户的提升提示行为：
  - 自动拒绝提升请求
  - 在安全桌面上提示凭据
  - **未配置**：提示凭据

- **应用安装的提升权限提示**：设置为“阻止”时，不会对应用程序安装包的权限提升进行检测和提示。 如果设置为“未配置”（默认设置）且某个应用程序安装包需要提升的权限，会提示用户提供管理用户名和密码。
- **在不使用安全桌面的情况下提示 UIA 提升权限**：设置为“启用”允许 UIAccess 应用在不使用安全桌面的情况下提示提升权限。 设置为“未配置”（默认设置）时，提升权限提示会使用安全桌面。

#### <a name="admin-approval-mode-settings"></a>管理员批准模式设置

- **内置管理员的管理员批准模式**：设置为“启用”，允许内置管理员帐户使用管理员批准模式。 任何需要提升权限的操作都将提示用户批准该操作。 “未配置”（默认设置）以完整的管理员权限运行所有应用。
- **以管理员批准模式运行所有管理员**：设置为“阻止”以禁用管理员批准模式和所有相关的 UAC 策略设置。 “未配置”（默认设置）会启用管理员批准模式。

### <a name="microsoft-network-client"></a>Microsoft 网络客户端

- **对通信进行数字签名(若服务器允许)**：确定 SMB 客户端是否协商 SMB 数据包签名。 设置为“未配置”或“启用”（默认设置）时，Microsoft 网络客户端会请求服务器在设置会话时运行 SMB 数据包签名。 如果在服务器上启用了数据包签名，则会协商数据包签名。 如果设置为“禁用”，SMB 客户端绝不会协商 SMB 数据包签名。
- **将未加密的密码发送到第三方 SMB 服务器**：设置为“启用”时，服务器消息块 (SMB) 重定向程序可以将明文密码发送给在身份验证期间不支持密码加密的非 Microsoft SMB 服务器。 设置为“未配置”（默认设置）时，会对密码进行加密。

### <a name="microsoft-network-server"></a>Microsoft 网络服务器

- **对通信进行数字签名(如果客户端允许)**：确定 SMB 服务器是否与请求它的客户端协商 SMB 数据包签名。 设置为“启用”时，Microsoft 网络服务器会根据客户端的请求协商 SMB 数据包签名。 也就是说，如果在客户端上启用了数据包签名，则会协商数据包签名。 未配置或禁用（默认设置）后，SMB 客户端绝不会协商 SMB 数据包签名。
- **对通信进行数字签名(始终)**：确定 SMB 服务器组件是否需要数据包签名。 设置为“启用”时，Microsoft 网络服务器不会与 Microsoft 网络客户端进行通信，除非该客户端接受 SMB 数据包签名。 设置为“未配置”或“禁用”（默认设置）时，客户端和服务器之间会协商 SMB 数据包签名。
